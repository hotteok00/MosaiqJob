Highlight Project 01  ·  협동로봇 · 모션 플래닝
MEDICREW AI 기반 협동 로봇 작업 어시스턴트
기간 2025.06.09 – 06.20 (12일) 팀 구성 4인 팀 역할 ROS2 제어 구조 설계 · 모션 플래닝 · Octomap 연동
병원 중앙공급실에서 협동로봇이 물품을 자동 수거하는 시스템. 음성 명령 → LangChain LLM → ROS2 Action → MoveIt2 모션 플래닝 → 실제 로봇 동작까지 한 파이프라인으로 연결했습니다.
상황 · 판단 · 행동 · 결과
상황
  • 협동로봇이 미리 정해진 경로로만 움직이면, 작업 중 놓인 장애물(기구, 물품)에 충돌 위험이 있었습니다.
  • 음성 명령을 로봇이 해석할 수 있는 구조화된 명령으로 실시간 변환하는 인터페이스가 필요했습니다.
판단
  • 정적 경로 계획 대신 Octomap 실시간 환경 맵 선택. 카메라 포인트 클라우드로 3D 맵을 매 주기마다 갱신하고, MoveIt2 플래너가 이를 참조해 충돌 없는 경로를 재계산하도록 구성했습니다. 대안으로 검토한 2D 격자 맵은 매니퓰레이터의 3차원 운동 공간을 커버할 수 없어 제외했습니다.
  • LangChain 기반 의도 파싱 선택. 규칙 기반 키워드 매칭은 자연어 변형에 취약했기 때문에, LLM이 음성 텍스트에서 동작 파라미터(위치, 대상)를 추출하고 ROS2 Action Goal 형식으로 직렬화하도록 설계했습니다.
행동
  • ROS2 노드 구조 설계: speech_node → llm_node → action_server → moveit_node 순서로 토픽·액션 인터페이스를 정의했습니다.
  • Octomap Server를 MoveIt2 Planning Scene과 연동 — 포인트 클라우드 업데이트 주기와 플래닝 요청 타이밍 간 레이스 컨디션을 콜백 동기화로 해결했습니다.
  • 장애물 감지 시 플래닝 재시도 횟수, 타임아웃, 재계획 트리거 조건을 파라미터화해 튜닝 가능하게 구성했습니다.
  • MongoDB에 작업 로그(명령·경로·실행 결과)를 저장해 이후 분석 가능하도록 했습니다.
결과
  • 실제 협동로봇에서 음성 명령 입력 후 물품 수거 동작까지 엔드투엔드 동작 확인.
  • 작업 경로에 장애물 배치 시 Octomap 기반 경로 재계획으로 충돌 없이 우회 동작 수행.
  • 다만, 음성 인식 오류율은 외부 STT 모델 의존성으로 인해 제어 불가한 변수로 남았습니다.

Highlight Project 02  ·  SLAM · Navigation · 시스템 설계
CODA SLAM 기반 교통사고 현장 보존 로봇
기간 2025.05.16 – 05.22 (7일) 팀 구성 7인 팀 역할 메인 서버 시스템 설계 · 개별 노드 구현
교통사고 발생 시 자율주행 로봇이 현장에 도착해 자동 촬영, 차량 통제, HTML 사고 리포트를 생성하는 시스템. SLAM으로 환경을 인식하고 Nav2로 목표 지점까지 자율 이동합니다.
상황 · 판단 · 행동 · 결과
상황
  • 7개 기능 노드(사고 감지, 이동, 촬영, 통제, 리포트 등)가 독립적으로 실행되면 타이밍 충돌로 데이터 정합성이 무너지는 문제가 있었습니다.
  • AutoSLAM 맵 생성 품질이 로봇 자율 이동 성공률을 직접 결정하는 구조였습니다.
판단
  • 분산 노드 대신 중앙 서버 상태 관리 아키텍처 선택. 각 노드가 이전 단계의 완료 신호를 받아야 실행되도록, 중앙 서버가 상태 머신(State Machine)으로 노드 실행 순서와 전이 조건을 통제하는 구조를 설계했습니다. ROS2 Service와 Action을 기능에 따라 구분해 사용했습니다.
  • AutoSLAM 파라미터 직접 튜닝. 기본 설정으로는 좁은 실내 공간에서 맵 왜곡이 발생했기 때문에 스캔 매칭 해상도와 루프 클로저 임계값을 조정해 맵 정확도를 높였습니다.
행동
  • 중앙 서버 상태 머신 설계: IDLE → DETECT → NAVIGATE → CAPTURE → CONTROL → REPORT 6단계 상태와 각 전이 조건을 정의했습니다.
  • AutoSLAM으로 사전 환경 맵 생성 후 Nav2 Lifecycle Manager를 통해 네비게이션 스택을 순서대로 활성화했습니다.
  • Nav2 DWB 플래너의 로컬 코스트맵 파라미터(inflation_radius, cost_scaling_factor)를 튜닝해 좁은 복도 통과 경로 생성에 성공했습니다.
  • YOLO로 차량·사람 감지 결과를 메인 서버에 전달하고, 감지 신뢰도 임계값 이하일 경우 재촬영 요청하는 로직을 추가했습니다.
  • 촬영 데이터, GPS 좌표, 감지 결과를 취합해 HTML 리포트를 자동 생성하는 노드를 구현했습니다.
결과
  • 실내 테스트 환경에서 사고 감지 → 현장 이동 → 촬영 → 리포트 생성 전 과정 자동 실행 확인.
  • 중앙 서버 상태 관리로 7개 노드의 실행 순서 충돌 없이 안정적으로 동작.
  • 7인 팀에서 메인 서버 설계를 단독으로 담당했고, 각 담당자의 노드와 인터페이스를 맞추는 협의 과정이 전체 일정의 절반을 차지했습니다.
